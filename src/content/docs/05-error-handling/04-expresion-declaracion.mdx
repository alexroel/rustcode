# Expresiones vs. Declaraciones

En Rust, uno de los conceptos fundamentales para entender cómo se estructura y se comporta el código es la diferencia entre **expresiones** y **declaraciones**. Ambos juegan un papel crucial en las funciones y pueden influir en cómo escribimos y organizamos nuestro código. Este artículo explora estas diferencias, su uso y ejemplos prácticos.

## ¿Qué son las Declaraciones?

Una **declaración** es una instrucción que establece algo, como la definición de una variable o una función. En otras palabras, las declaraciones no devuelven un valor, sino que crean o definen algo en el contexto del programa. Por ejemplo, una declaración de función define la función y sus parámetros, pero no produce un valor en sí misma.

### Ejemplo de Declaración

Aquí hay un ejemplo de una declaración de función en Rust:

```rust
fn saludar() {
    println!("¡Hola, mundo!");
}
```

En este caso, `fn saludar()` es una declaración que define la función `saludar`. Cuando se llama a esta función, imprime un mensaje en la consola, pero la declaración en sí no devuelve ningún valor.

## ¿Qué son las Expresiones?

Una **expresión** es una combinación de valores y operadores que se evalúan para producir un valor. A diferencia de las declaraciones, las expresiones sí devuelven un resultado. En Rust, muchas construcciones del lenguaje, como las operaciones aritméticas y las llamadas a funciones, son expresiones.

### Ejemplo de Expresión

Considera el siguiente ejemplo:

```rust
let suma = 5 + 3;
```

Aquí, `5 + 3` es una expresión que se evalúa para producir el valor `8`, que luego se asigna a la variable `suma`.

## Diferencias Clave entre Expresiones y Declaraciones

1. **Valor Devuelto:**
   - **Declaraciones:** No devuelven un valor. Su propósito es definir algo.
   - **Expresiones:** Siempre devuelven un valor como resultado de su evaluación.

2. **Uso en Funciones:**
   - Las funciones en Rust pueden contener tanto declaraciones como expresiones. Las declaraciones se utilizan para definir variables o funciones, mientras que las expresiones se utilizan para realizar cálculos o evaluaciones.

## Ejemplo de Función que Combina Declaraciones y Expresiones

Veamos una función que utiliza tanto declaraciones como expresiones:

```rust
fn calcular_area(base: f64, altura: f64) -> f64 {
    let area = base * altura; // Aquí, `base * altura` es una expresión.
    area // Esto es una expresión que devuelve el valor de `area`.
}

fn main() {
    let base = 5.0;
    let altura = 3.0;
    let resultado = calcular_area(base, altura);
    println!("El área es: {}", resultado);
}
```

### Desglose del ejemplo:

1. **Declaraciones:**
   - `let area = base * altura;`: Esta línea define una variable llamada `area`, que se calcula usando una expresión.
   - `fn calcular_area(base: f64, altura: f64)`: Esta línea es una declaración que define la función `calcular_area`.

2. **Expresiones:**
   - `base * altura`: Esta parte es una expresión que se evalúa para calcular el área.
   - `area`: Al final de la función, `area` se considera una expresión que devuelve el valor calculado.

### Salida del programa

Si ejecutas el programa, obtendrás la siguiente salida:

```
El área es: 15
```

## Conclusión

Entender la diferencia entre expresiones y declaraciones es crucial para escribir código efectivo en Rust. Mientras que las declaraciones establecen y definen cosas en tu programa, las expresiones permiten realizar cálculos y devolver valores. Esta distinción no solo es importante para la sintaxis, sino también para la lógica de programación y la estructura del código.
